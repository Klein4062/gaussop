# 质量改进诉求流程表单

用 Python(openpyxl)生成「质量改进诉求」全流程表单的工作集。覆盖 **提出 → 评审 → 改进项分析 → 闭环 → 验收** 全链路,并将系统能自动生成/继承的字段单独标注,明确「哪些由用户填、哪些由系统填」。

## 仓库结构

| 文件 | 作用 | 产物 |
|---|---|---|
| `build_quality_forms.py` | **正式表单**生成器(可视化版式) | `质量改进诉求流程表单.xlsx`(10 个 Sheet) |
| `build_field_specs.py` | **字段规格审核表**生成器(五维字段清单) | `质量改进诉求表单-字段规格审核.xlsx`(8 个 Sheet) |
| `质量改进诉求流程表单.xlsx` | 正式表单产物(可直接使用/审核) | — |
| `质量改进诉求表单-字段规格审核.xlsx` | 字段定义审核表(定稿字段口径) | — |

> 两份产物口径一致:审核表用于评审字段定义,正式表单是落地版式。改字段先改审核表,确认后同步回正式表单。

## 业务流程

```
① 提出(关联运维系统单号)
   ➜ ② 评审(通过 → 指派责任人;不通过 → 打回提出)
       ➜ ③ 改进项分析(接纳 → 定闭环方法:问题单/需求 → 挂闭环单号,校验合法性;不接纳 → 打回评审)
           ➜ ④ 闭环(问题单路径 / 需求路径,各关注进展与效果自测;含SLA时间)
               ➜ ⑤ 验收(谁提出谁验收,不可转单,通过即完成)
```

- **单链流程**(不拆子项):一张主诉求单即一项改进措施,整链流转。
- 评审完成前,提出单可由提出人编辑修改;评审不通过打回提出、分析不接纳打回评审。
- **责任人链**:评审填写 → 分析继承 → 闭环继承。
- 闭环分两条路径:**问题单闭环**(缺陷/故障纠正预防)、**需求闭环**(新功能/优化上线)。
- **谁提出谁验收**:验收人=改进提出人,锁定不可转单;通过即完成(无独立归档环节)。
- 改进项分析单含「接纳版本」;分析与闭环阶段均含「进展子项」(序号/进展说明/提交时间/提交人),可逐条单独提交,作为外部审视进展的依据。
- 闭环自测 / 验收结论支持**富文本与粘贴图片**;审批人为固定下拉名单。

## 表单清单

| Sheet | 表单 | 字段数 |
|---|---|---|
| 1-提出 | 质量改进诉求单(关联运维系统单号) | 8 |
| 2-评审 | 评审记录单(通过→指派责任人) | 7 |
| 3-分析 | 改进项分析单 | 15 |
| 4-1 | 问题单闭环单(含SLA/进展子项) | 13 |
| 4-2 | 需求闭环单(含SLA/进展子项) | 13 |
| 5 | 验收单(谁提出谁验收) | 7 |

另有:`0-流程总览`、`1.5-改进诉求台账(中枢)`、`6-填写约定与编号规则`。

## 填写类型约定

正式表单按输入区底色区分;审核表用五维标注。

| 类型 | 正式表单底色 | 含义 |
|---|---|---|
| 用户填写 | 白底 | 用户录入(必填 / 选填 / 条件必填) |
| 系统生成 | 绿底(带「系统生成」) | 系统自动生成/继承,用户免填(编号、状态、关联、日期、责任人继承、提交人等) |
| 预填可改 | 紫底(带「预填·可改」) | 系统预填默认值,用户可修改 |

## 编号规则

| 编号 | 格式 |
|---|---|
| 主诉求单号 | `QI-YYYY-NNN` |
| 关联运维系统单号 | `YW-YYYY-NNN`(运维系统原始单号) |
| 分析单号 | `AN-YYYY-NNN` |
| 问题单号 | `PC-YYYY-NNN` |
| 需求单号 | `RC-YYYY-NNN` |
| 验收单号 | `AC-YYYY-NNN` |

## 快速开始

```bash
pip install openpyxl
python3 build_quality_forms.py     # 生成 正式表单 xlsx
python3 build_field_specs.py       # 生成 字段规格审核 xlsx
```

用 Apple Numbers / Excel / WPS 均可打开生成的 `.xlsx`。

## 约定

- 改字段先改 `build_field_specs.py`(审核表),定稿后同步到 `build_quality_forms.py`(正式表单)。
- 表格行(进展子项、改进诉求台账)不足时,直接在下方插入行即可。
- 枚举字段已挂下拉(状态、评审结果、是否接纳、闭环方法、验收是否通过、评审人等)。
